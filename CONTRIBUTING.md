\# Contributing to AXON Protocol



Thank you for your interest in AXON. We believe digital truth requires a collective effort. To maintain the integrity of the protocol, every contribution must adhere to our core principles.



\## 🛡️ Non-Negotiable Pillars

Every Pull Request will be evaluated based on these criteria:

1\. \*\*Absolute Neutrality:\*\* The code must never favor a specific provider or geographic area.

2\. \*\*Rust Efficiency:\*\* We do not accept heavy dependencies or code that compromises "zero-lag" performance.

3\. \*\*Privacy by Design:\*\* AXON signs metadata; it does not track users. Any attempt to introduce tracking or surveillance will be rejected.

4\. \*\*L3 Compatibility:\*\* Modifications must be optimized for the Layer 3 architecture with zero-gas cost.



\## 🏛️ Provider Accession (Expandable Governance)

AXON is an inclusive protocol by design. Any AI Provider can apply to join the Board of Founders:

\- \*\*Admission Criteria:\*\* Joining the Board requires a \*\*unanimous vote\*\* from current members.

\- \*\*Transparency of Denial:\*\* To prevent the protocol from becoming a barrier to progress or a tool for competitive exclusion, \*\*every negative vote must be formally justified with technical reasoning\*\*.

\- \*\*Common Security:\*\* A denial is only legitimate if based on documented technical incompatibility or proven risks to the global security of the protocol. The goal is to keep everyone safe, not to limit access.



\## 🤝 Review Process

\- \*\*Discussion:\*\* For substantial changes, please open an Issue first to discuss the idea.

\- \*\*Unanimous Vote:\*\* Changes to the core protocol or Governance Smart Contracts require unanimous approval from the Board.

\- \*\*Code Quality:\*\* All contributions must follow Rust standards (`cargo fmt`) and include unit tests.



\## ⚖️ Code of Conduct

AXON is a technical and diplomatic project. We maintain an environment of mutual respect, focused exclusively on cybersecurity and media integrity.
